ChatVibes TTS Bot Documentation

ChatVibes is currently invite-only. Request access.

Commands

All TTS commands are prefixed with !tts.
For example: !tts status
Click any command row below to expand and see usage details.
Broadcasters and viewers can also configure settings using the ChatVibes Web Dashboard.
Viewers can set their own TTS preferences using the web UI by sending !myvoice in chat.

Text-To-Speech Commands

Command Description
General
!tts status Gets the current status of the TTS application for the channel.▼
!tts status

Notes: Displays engine state, mode, queue length, default voice, pitch, speed, emotion, and language boost settings.

Permission: Everyone
!tts voices Provides a link to the documentation section for available TTS voices.▼
!tts voices

Permission: Everyone
!tts languageslist Provides a link to the documentation section for available TTS language boost options.▼
!tts languageslist

Permission: Everyone
!tts commands
(Alias: !tts help)		 Provides a link to the full list of !tts subcommands.▼
!tts commands !tts help

Permission: Everyone
!tts prefs
(Aliases: !tts preferences, !tts settings) 		Sends you a short-lived link to your personal TTS settings page.▼
!tts prefs !tts preferences !tts settings
  • Configure your own voice, pitch, speed, emotion, and language on a per-channel basis.
  • Includes a red-outlined "Danger Zone" where you can add yourself to the channel's TTS and/or Music ignore lists (only a moderator can undo this).
  • The bot posts the link publicly in chat; it is signed for you only and expires after first access or 10 minutes. The page is pre-filled with the current channel.

Permission: Everyone
!myvoice Sends you a short-lived link to your personal TTS settings page (equivalent to !tts prefs).▼
!myvoice

Notes: Same as !tts prefs. The bot posts the link publicly in chat; it is signed for you only and expires after first access or 10 minutes. The page is pre-filled with the current channel.

Permission: Everyone
Engine & Mode Control (Moderator Only)
!tts on
(Alias: !tts enable) 		Enables the TTS engine. Messages and events may be spoken based on the current mode.▼
!tts on !tts enable

Permission: Moderator
!tts off
(Alias: !tts disable) 		Disables the TTS engine entirely. No messages or events will be spoken.▼
!tts off !tts disable

Permission: Moderator
!tts mode [all|command|bits_points_only] Toggles the TTS mode.▼
!tts mode all (Speak all chat messages & events; this is the default mode) !tts mode command (Only speak on command or for events) !tts mode bits_points_only (Only speak Bits cheers & Channel Point redemptions) !tts mode bits (Alias for bits_points_only) !tts mode points (Alias for bits_points_only) !tts mode (Displays current mode)

Notes: all: All chat messages (not from ignored users or commands) and enabled events will be spoken, subject to the !tts permission setting.
command: Only messages triggered by specific TTS commands (like !tts <message>) or enabled events will be spoken. Regular chat is ignored.
bits_points_only: Only Bits cheers and Channel Point redemptions are read aloud. Regular chat and commands are ignored.

Permission: Moderator
!tts permission [everyone|all|mods|vip] Sets a filter on who can trigger TTS when the bot is in all mode.▼
!tts permission everyone !tts permission mods !tts permission vip !tts permission (displays the current permission filter)

Notes: everyone or all: Any user's chat message can be spoken. (Default)
mods: Only messages from moderators and the broadcaster will be spoken.
vip: Only messages from VIPs, moderators, and the broadcaster will be spoken.
This does not affect command mode.

Permission: Moderator
!tts bits [on|off|min <amount>] Configures the Bits-for-TTS feature, allowing channels to require a minimum Cheer amount to trigger TTS.▼
!tts bits on !tts bits off !tts bits min 100 !tts bits (displays current Bits-for-TTS status)

Description:

  • on: Enables Bits-for-TTS mode. Only messages with a sufficient cheer will be read.
  • off: Disables Bits-for-TTS mode.
  • min <amount>: Sets the minimum number of Bits required (e.g., min 100).

Permission: Moderator
Channel Points → TTS (Web Dashboard Only)
Channel Points → TTS Create a custom Twitch Channel Point reward for viewers to redeem with a message to be read aloud.▼

Note: This feature is configured exclusively through the ChatVibes Web Dashboard under the "Channel Points → TTS" section.

How It Works:

  1. Streamer Setup: Enable the feature in the dashboard.
  2. Reward Creation: The bot automatically creates a custom Channel Point reward on your Twitch channel.
  3. Viewer Redemption: Viewers redeem the reward with a custom message, which gets read aloud.
  4. Content Policy: Built-in filtering (length limits, link blocking, banned words).

Configuration Options:

  • Basic Settings: Reward title, cost (points), prompt text, skip queue option
  • Advanced Settings: Global cooldown, per-stream limit, per-user limit
  • Content Policy: Min/max message length, block links, banned words list

Skip the Reward Queue:

  • Default Setting: This option is enabled by default for immediate TTS playback.
  • When Enabled: Redemptions are processed immediately and do not appear in your Twitch "Reward Requests" queue.
  • When Disabled: Redemptions appear in your Twitch dashboard for manual approval before TTS plays.
  • Use Case: Enable for instant TTS, disable for moderation control.

TTS Modes: Works with all TTS modes (All Messages, Commands Only, Bits/Points Only).

Requirements: Must grant channel:manage:redemptions and channel:read:redemptions OAuth scopes during initial dashboard setup.

Permission: Streamer/Broadcaster (via Web Dashboard)
Queue Management (Moderator Only)
!tts pause Pauses the processing of the TTS event queue.▼
!tts pause

Notes: New messages/events will still be added to the queue but won't be spoken until resumed.

Permission: Moderator
!tts resume Resumes processing of the TTS event queue if it was paused.▼
!tts resume

Permission: Moderator
!tts clear Clears all pending messages and events from the TTS queue.▼
!tts clear

Notes: This command only removes messages waiting in the queue. It does not stop audio that is currently playing or actively being generated by the TTS engine.

Permission: Moderator
!tts stop Stops the currently playing or generating TTS audio.▼
!tts stop

Notes:

  • If TTS is currently speaking or generating audio, this command will attempt to stop it.
  • Any user can stop a message if it was triggered by their own chat message.
  • Moderators and the broadcaster can stop any TTS audio.

Permission: Everyone for their own messages, Moderator for others
User & Event Preferences
!tts voice <voice_id|reset> Allows a user to set their preferred voice for messages they send.▼
!tts voice Friendly_Person !tts voice reset !tts voice (displays current personal preference)

Notes: Use reset to revert to the channel's default voice. Use !tts voices to get a link to available voice IDs.

Permission: Everyone (for their own preference)
!tts emotion <emotion_name|reset|auto> Allows a user to set their preferred emotion for their messages.▼
!tts emotion happy !tts emotion reset !tts emotion (displays current personal preference)

Notes: Valid emotions: neutral, happy, sad, angry, fearful, disgusted, surprised. Use reset for the channel default.

Permission: Everyone (for their own preference)
!tts language <language|reset>
(Alias: !tts lang)		 Sets your preferred language boost for TTS.▼
!tts language English !tts lang Japanese !tts language reset !tts language (displays current personal preference)

Notes: Affects how speech is interpreted and generated. Use reset or auto to use the channel's default. See Language Boost section or use !tts languageslist for available options.

Permission: Everyone (for their own preference)
!tts pitch <value|reset> Sets your personal TTS pitch.▼
!tts pitch 2 !tts pitch -3 !tts pitch reset !tts pitch (displays current personal preference)

Notes: Value must be an integer between -12 and 12 (0 is normal). Use reset for the channel default.

Permission: Everyone (for their own preference)
!tts speed <value|reset> Sets your personal TTS speed.▼
!tts speed 1.2 !tts speed 0.8 !tts speed reset !tts speed (displays current personal preference)

Notes: Value must be a number between 0.5 and 2.0 (1.0 is normal). Use reset for the channel default.

Permission: Everyone (for their own preference)
!tts ignore <username>
!tts ignore add <username>		 For any user: Add yourself to the TTS ignore list for the channel. Your messages will not be spoken.
For Moderators or Broadcaster: Add any specified Twitch user to the TTS ignore list for the channel.
▼
Usage:
  • !tts ignore yourusername (if you want to ignore yourself)
  • !tts ignore add SomeOtherUser (if you are a mod/broadcaster)

Permission: Everyone (to add themselves), Moderator (to add others)
!tts ignore del <username>
(Aliases: delete, rem, remove)		 Removes the specified Twitch user from the TTS ignore list.▼
!tts ignore del SomeUser

Permission: Moderator
!tts ignored Lists all users currently on the TTS ignore list for the channel.▼
!tts ignored

Permission: Moderator
!tts events [on|off] Toggles whether Twitch events (like subscriptions, cheers, raids, etc.) are announced by TTS.▼
!tts events on !tts events off !tts events (displays current event announcement status)

Permission: Moderator
Channel-Wide Default Configuration (Moderator Only)
!tts defaultvoice <voice_id|reset> Sets the default TTS voice for the channel.▼
!tts defaultvoice Friendly_Person !tts defaultvoice reset !tts defaultvoice (displays current channel default)

Notes: Use reset to revert to the system default. Use !tts voices for a link to voice IDs.

Permission: Moderator
!tts defaultemotion <emotion_name|reset> Sets the default TTS emotion for the channel.▼
!tts defaultemotion happy !tts defaultemotion reset !tts defaultemotion (displays current channel default)

Notes: Valid emotions: neutral, happy, sad, angry, fearful, disgusted, surprised. Use reset for system default (neutral).

Permission: Moderator
!tts defaultlanguage <language|reset> Sets the default TTS language boost for the channel.▼
!tts defaultlanguage English !tts defaultlanguage reset !tts defaultlanguage (displays current channel default)

Notes: Use reset or auto to use the system default. See Language Boost section or use !tts languageslist for options.

Permission: Moderator
!tts defaultpitch <value|reset> Sets the default TTS pitch for the channel.▼
!tts defaultpitch 2 !tts defaultpitch reset !tts defaultpitch (displays current channel default)

Notes: Value must be an integer between -12 and 12 (0 is normal). Use reset for system default (0).

Permission: Moderator
!tts defaultspeed <value|reset> Sets the default TTS speed for the channel.▼
!tts defaultspeed 1.2 !tts defaultspeed reset !tts defaultspeed (displays current channel default)

Notes: Value must be a number between 0.5 and 2.0 (1.0 is normal). Use reset for system default (1.0).

Permission: Moderator
Direct TTS
!tts <message>
(Alias: !tts say <message>)		 Immediately enqueues the provided message for TTS. This works in both all and command modes. Uses the requesting user's preferences (or channel defaults) for voice, pitch, speed, and emotion.▼
!tts Welcome everyone to the stream! !tts say Welcome everyone!

Permission: Everyone
!tts say <message> (Deprecated: see above)

Music Generation Commands

All music generation commands are prefixed with !music
For example: !music 90s soulful house
Click any command row below to expand and see usage details.

Music Generation Commands

Command Description
!music <prompt> Generates a short piece of music based on your prompt.▼
!music epic orchestral battle theme

Permission: Configurable by mods (default: everyone, can be mods only). User must not be on music ignore list.
!music on
!music off		 (Mod only) Enables or disables the music generation feature for the channel. ▼
!music on !music off

Permission: Moderator
!music mode <all|mods> (Mod only) Sets who can use the !music <prompt> command.
all: Everyone can generate music.
mods: Only moderators and the broadcaster can generate music. ▼
!music mode all !music mode mods

Permission: Moderator
!music bits [on|off|min <amount>] (Mod only) Configures Bits-for-Music. When enabled, users must cheer with their prompt to generate music. ▼
!music bits on !music bits off !music bits min 100 !music bits (displays current status)

Description:

  • on: Enables Bits-for-Music mode.
  • off: Disables Bits-for-Music mode.
  • min <amount>: Sets the minimum number of Bits required (e.g., min 100).

Permission: Moderator
!music help Provides a list of all music subcommands and their usage.▼
!music help

Notes: Lists all !music subcommands and their usage, similar to this documentation.

Permission: Everyone
!music status Shows the current status of music generation (enabled/disabled, mode, queue length).▼
!music status

Permission: Everyone
!music clear (Mod only) Clears all pending music generation requests from the queue.▼
!music clear

Permission: Moderator
!music ignore <username>
!music ignore add <username>		 For any user: Add yourself to the music ignore list. Your !music <prompt> requests will be ignored.
For Moderators or Broadcaster: Add any specified Twitch user to the music ignore list.
▼
Usage:
  • !music ignore yourusername (if you want to ignore yourself)
  • !music ignore add SomeOtherUser (if you are a mod/broadcaster)

Permission: Everyone (to add themselves), Moderator (to add others)
!music ignore del <username>
(Aliases: delete, rem, remove) 		(Mod only) Removes the specified Twitch user from the music ignore list. ▼
!music ignore del SomeUser

Permission: Moderator
!music ignored (Mod only) Lists all users currently on the music ignore list for the channel. ▼
!music ignored

Permission: Moderator

Language Boost

The language_boost feature helps the TTS model better understand and pronounce words from specific languages or dialects. This is particularly useful in multilingual chats or when a specific language's nuances are important for clear TTS.

You can set a personal language boost preference using !tts language <language_name> or moderators can set a channel default with !tts defaultlanguage <language_name>.

Available Language Boost Options

The following values are accepted:

  • auto
  • Arabic
  • Chinese
  • Chinese,Yue
  • Czech
  • Dutch

Using auto will rely on the model's automatic language detection. English is often a good default for primarily English-speaking chats. Choosing a specific language can significantly improve pronunciation for that language.

Web Dashboard

The ChatVibes Web Dashboard provides streamers with a comprehensive interface to configure all TTS and music settings. Access it by logging in with your Twitch account.

Authentication Modes

ChatVibes offers two ways for streamers to connect:

  • Bot-Free Mode (Stealth): The bot operates in "stealth" mode. It can read chat, listen for commands, and announce events (Subs, Bits, Follows) via EventSub, but it cannot send messages to chat or appear in the user list. This is ideal for streamers who want TTS without adding another bot to their chat list.
  • Chatbot Mode (Full): The bot has full functionality, including the ability to reply in chat (e.g., confirming commands) and be managed as a moderator.

Available Dashboard Settings

  • Engine Control: Enable/disable TTS, announce events, set TTS mode and permissions
  • Viewer Preferences: Allow or force channel default voice for all messages
  • URL Reading: Read full URLs or just domain names
  • Bits → TTS: Require minimum bits for TTS messages
  • Channel Points → TTS: Create and configure a custom Channel Point reward (see section above)
  • Music Generation: Enable/disable music, set permissions, require bits
  • Ignore Lists: Manage TTS and music ignore lists with add/remove interface
  • Default Voice Settings: Set channel defaults for voice, volume, emotion, pitch, speed, language, and text normalization
  • Voice Calibration: Set specific volume overrides for individual voices to normalize audio levels
  • OBS Setup: Get your unique secure TTS URL and detailed setup instructions
  • Voice Preview: Test your current voice settings before going live

Voice Calibration

Some voices may be naturally louder or quieter than others. The Voice Calibration feature allows you to set specific volume levels for individual voices, ensuring a consistent audio experience across your stream.

  • Per-Voice Volume: Select any voice and assign a custom volume multiplier (0.1x to 10.0x). (Warning: Values over 1.0 may be loud)
  • Auto-Apply: When a user selects a calibrated voice via !myvoice, the custom volume is automatically applied.
  • Overrides: Calibrated volumes take precedence over the channel's default volume setting for that specific voice.

OBS Browser Source Setup

Step 1: Add Browser Source in OBS

  1. In OBS, click the "+" under Sources
  2. Select "Browser"
  3. Name it (e.g., "ChatVibes TTS")
  4. Configure:
    • URL: Paste your unique URL from the dashboard
    • Width/Height: 300x100 (doesn't matter, it's audio-only)
    • ✓ Control audio via OBS (must be checked)
    • Uncheck "Shutdown source when not visible" for persistent audio

Step 2: Enable Audio Monitoring

  1. Find the audio mixer in OBS (bottom panel)
  2. Click the gear (⚙️) next to "ChatVibes TTS" → "Advanced Audio Properties"
  3. Set "Audio Monitoring" to:
    • Monitor and Output - You AND stream hear it
    • Monitor Only - Only you hear it (for testing)

Important: Make sure the source is unmuted in your audio mixer and volume is set appropriately.

Troubleshooting

  • Bot not responding to commands? Make sure the bot is a moderator in your channel. You can do this by typing /mod chatvibesbot in your Twitch chat. In many cases, the bot may need moderator permissions to read messages and respond to commands.
  • Bot not speaking? Make sure the bot is enabled in your channel. You can do this by typing !tts on in your Twitch chat. Also, check the current mode !tts mode and permissions !tts permission.
  • Bot not speaking in a specific language, or reading numbers in Chinese? Make sure the language boost is set to the correct language. You can do this by typing !tts language <language_name> for yourself or !tts defaultlanguage <language_name> for the channel in your Twitch chat. You can check the current language boost using !tts defaultlanguage.
  • TTS generation taking too long or failing? ChatVibes now uses Wavespeed AI with sync mode enabled for the lowest latency. If you experience delays, check your internet connection and try using !tts stop to cancel any stuck generation. The bot automatically retries failed requests.
  • Voice not found error? Make sure you're using the correct voice ID from the Voices section. Voice IDs are case-sensitive and must match exactly. Try !tts voices to get a link to available voices.
  • "Voice access denied" error? Some voices (particularly certain Moss audio effects) may require special access permissions on Wavespeed AI. If you receive this error, try using a different voice from the available voices list. Most voices work with standard access.
  • "Invalid voice" error? The voice ID you're using is not available or doesn't exist. Make sure you're using the correct voice ID from the Voices section. Voice IDs are case-sensitive and must match exactly.
  • Emotion not working? Note that "auto" emotion is no longer supported. Use neutral, happy, sad, angry, fearful, disgusted, or surprised instead.
